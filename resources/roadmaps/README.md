## Open Source Contribution Guide

Getting into GSoC, Outreachy, or MLH? Start here:

1. Pick a project you use (check [goodfirstissue.dev](https://goodfirstissue.dev))
2. Read the contribution guide and set up the dev environment
3. Fix a "good first issue" — focus on docs, tests, or small bugs
4. Build relationships with maintainers before applying to programs
5. Apply with a strong proposal that shows you understand the codebase
