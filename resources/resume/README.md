## ✅ ATS Checklist

Your resume will often be screened by Applicant Tracking Systems (ATS) before a human reads it.

- [ ] Use a single-column layout for maximum ATS compatibility
- [ ] Avoid tables, headers/footers, or multi-column formats
- [ ] Use standard section names: **Education**, **Experience**, **Projects**, **Skills**
- [ ] Include keywords from the job description
- [ ] Use `.pdf` format unless `.docx` is specifically requested
- [ ] Keep it to **1 page** for fresher / internship applications

## 💡 Pro Tips

### For Freshers / Students
- Lead with **Projects** if you have limited work experience
- Quantify everything: "Reduced API latency by 40%" > "Improved API performance"
- GitHub link > LinkedIn for tech roles — keep your GitHub active
- List only skills you can confidently discuss in an interview

### For Internships
- Tailor your resume for each application (especially the skills section)
- Highlight coursework relevant to the role
- Include competitive programming rankings (LeetCode, Codeforces)
- Mention open-source contributions prominently
